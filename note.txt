setup yang diperlukan:
- setup objek ticket
- setup objek ticket bisa ada child and parent
- setup type penyebab khusus utk objek bertype error, misal type penyebab: human error, system error. ini user harus bisa custom
- title utk objek ticket tiap level (ex: modul, submodul, subsubmodul[ini default jika tidak ada title])
- [opsional] tampilkan statistik ticket user, pilih jenis diagram



catatan:
utk objek yang ingin pindah parent, new parent harus sama level nya dengan old parent
kalo pun new parent tidak sama dengan old parent level nya, maka objek(yg ingin pindah parent) harus diubah level nya dan level child nya(jika ada) menyesuaikan dengan new parent
